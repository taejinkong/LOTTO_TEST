# Lotto Analyzer

대한민국 로또 1등 번호 CSV를 바탕으로 통계와 조건부 연계 규칙을 분석하는 도구입니다.

현재 데이터 파일:

- `lotto_winners_2020_2026.csv`
- `로또 회차별 당첨번호_20260620124844.xlsx`

## 당첨번호 업데이트 (동행복권 공식 API)

새 회차만 받아 CSV에 이어붙이고, 앱 데이터(`app/data.js`)까지 자동 갱신합니다.

```bash
python3 update_lotto_csv.py          # 새 회차 추가 + app/data.js 재생성
```

엔드포인트: `dhlottery.co.kr/lt645/selectPstLt645InfoNew.do` (공식 JSON).

> ⚠️ 동행복권은 **클라우드/데이터센터 IP나 자동화 요청을 차단**(홈·errorPage로
> 리다이렉트)합니다. 차단되면 CSV는 그대로 두고 안내만 출력합니다.
> **일반 가정용 네트워크의 본인 PC에서 직접 실행**하면 보통 정상 동작합니다.

차단으로 받아오지 못할 때는 사이트에서 번호를 보고 한 줄을 직접 추가할 수 있습니다.

```bash
# '회차,추첨일,n1~n6,보너스[,1등당첨자수,1인당당첨금,총판매금액]'
python3 update_lotto_csv.py --add '1230,2026-06-27,3,8,9,22,28,42,45,16,1771357196,120177677000'
```

전체를 다시 받으려면 `--mode rebuild` 를 씁니다.

## 엑셀 데이터 반영

동행복권 회차별 당첨번호 엑셀 파일을 분석용 CSV로 변환합니다.

```bash
python3 import_lotto_excel.py '로또 회차별 당첨번호_20260620124844.xlsx'
```

기본 범위는 `2014-07-12`부터 `2026-06-13`까지이며, `606회~1228회`를 추출합니다.

엑셀 파일에 추첨일과 총판매금액이 없기 때문에:

- `추첨일`은 로또 1회 기준 매주 토요일 규칙으로 계산합니다.
- `총판매금액(원)`은 `0`으로 채웁니다.

## 실행

```bash
python3 lotto_analyzer.py
```

상위 규칙 개수나 필터 기준을 조정할 수 있습니다.

```bash
python3 lotto_analyzer.py --limit 50 --min-support 15 --min-confidence 0.5 --min-lift 1.5
```

## 시나리오별 후보 생성

단일 점수 상위 조합만 뽑으면 희귀하지만 실제로 발생하는 패턴이 탈락할 수 있습니다. 그래서 후보를 여러 시나리오로 나눠 생성합니다.

```bash
python3 predict_lotto.py --per-scenario 5
```

현재 시나리오:

- `balanced`: 합계 100~159, 연속번호 1쌍 이하
- `high_sum`: 합계 160 이상
- `high_start`: 첫 수 21 이상
- `consecutive`: 연속번호 2쌍 이상
- `high_band`: 31~45 번호 3개 이상

기본 모델 `integrated_rules_v2`는 분석 가능한 규칙을 모두 0~1로 정규화해 사용합니다.

- 기본 패턴(홀짝·번호대·끝수·합계·저/고번호·연속·AC·간격·첫 수) × `0.03`
- 전이 패턴(이월수·이웃수) × `0.02`
- 조건부 연계 규칙 × `0.04`
- 개별 번호(전체·최근 빈도와 미출현) × `0.03`
- 번호쌍 전체·최근 동반 출현 × `0.56`
- 번호별 출현주기 × `0.32`
- 직전과 후보가 모두 희귀 홀짝일 때 `0.35` 감점

30회 표본 백테스트에서 통합 v2는 평균 백분위 `42.2%`, 기존 페어+주기 v1은
`40.8%`였습니다. 따라서 규칙 통합은 분석 범위 확장 기능이며 성능 개선으로
해석하지 않습니다. 웹앱에서는 두 모델을 선택할 수 있고 Python 생성기의 기본값은
통합 v2입니다.

웹앱은 다음 검증·기록 기능도 제공합니다.

- 같은 설정과 조합을 다시 만들 수 있는 문자열 시드
- 상위 점수 조합끼리 번호가 과도하게 겹치지 않게 하는 포트폴리오 분산
- 합계·홀짝 극단·끝수 합계·3연속·번호대·직전 번호 제한을 기본값이 꺼진 실험 필터로 제공
- 각 실험 필터의 과거 당첨조합 생존율과 무작위 후보 생존율 표시
- 순수 무작위의 기대 백분위 `50%` 및 30회 평균의 이론적 변동 범위 표시
- 회차·데이터 기준·모델·조건·시드·조합을 SHA-256 영수증과 함께 저장하는 로컬 예측 원장
- 새 추첨 데이터 반영 후 보너스 번호를 포함한 실제 1~5등 자동 판정

최근 최다 출현번호를 모든 조합에 강제로 넣거나, 전형적인 형태의 필터를 기본으로
적용하지 않습니다. 규칙별 통과율은 예측력이 아니라 제외 위험을 설명하는 지표입니다.

## 1228회 백테스트

`892회~1227회`까지만 학습하고 `1228회`를 예측했을 때의 차이를 확인합니다.

```bash
python3 backtest_1228.py --target-round 1227 --mode full --per-scenario 50
```

`--target-round`로 예측 점검 회차를 바꿀 수 있습니다. 백테스트는 전체 조합을 한 번만 순회하면서 기존 단일 점수, 개선 포트폴리오 점수, Top-K 포함 여부, 시나리오별 실제 1등 조합 포함 여부를 함께 계산합니다.

`full` 모드에서는 시나리오별 후보를 아래 쿼터로 합쳐 최종 Top 50 포트폴리오를 만듭니다. 평가는 실제 1등 조합 포함 여부뿐 아니라 Top 50 안에 1등, 2등, 3등 조합이 각각 몇 개 있는지를 기준으로 봅니다.

- `balanced`: 25개
- `high_sum`: 8개
- `high_band`: 7개
- `high_start`: 5개
- `consecutive`: 5개

등수 판정:

- `1등`: 실제 당첨번호 6개 일치
- `2등`: 실제 당첨번호 5개 + 보너스 번호 일치
- `3등`: 실제 당첨번호 5개 일치

모드:

- `fast`: 전체 조합 순위와 Top-K 포함 여부만 계산합니다.
- `full`: `fast` 결과에 시나리오별 Top-N 포함 여부까지 계산합니다.
- `sample`: 무작위 표본으로 빠르게 점검합니다.

예:

```bash
python3 backtest_1228.py --target-round 1227 --mode sample --sample-size 50000
python3 backtest_1228.py --target-round 1227 --mode full --top-k 50,300,1000,10000,100000
python3 backtest_1228.py --target-round 1227 --mode full --show-contribution
```

## 조건부 연계 규칙

규칙은 다음 형태로 출력됩니다.

```text
IF 조건1 AND 조건2 THEN NEXT 목표패턴 = 적중/발생, conf 신뢰도, base 기본확률, lift 리프트
```

- `support`: 조건이 발생한 횟수입니다. 출력에서는 `적중/발생`의 분모입니다.
- `confidence`: 조건이 발생했을 때 목표 패턴이 나온 비율입니다.
- `base`: 목표 패턴의 전체 기본 발생률입니다.
- `lift`: 기본 발생률 대비 조건부 확률이 얼마나 강한지입니다.

예:

```text
IF OE=3:3 AND LOW_1_10=0 THEN NEXT OE=3:3
```

직전 회차가 홀짝 `3:3`이고 1~10 번호가 없을 때, 다음 회차 홀짝 패턴이 `3:3`이 되는지를 검증합니다.

## 주요 지표

- `OE`: 홀짝 패턴입니다. 예: `OE=3:3`
- `COLOR`: 번호대 그룹입니다. `1~10`, `11~20`, `21~30`, `31~40`, `41~45` 순서입니다.
- `LOW_1_10`: 1~10 번호 개수입니다.
- `HIGH_31_45`: 31~45 번호 개수입니다.
- `SUM`: 번호 6개 합계 구간입니다.
- `CONSEC`: 연속번호 쌍 개수입니다.
- `AC`: AC값 구간입니다.
- `GAP`: 번호 간격의 밀집도입니다.
- `FIRST`: 첫 수 구간입니다.
- `CARRY`: 직전 회차에서 이월된 번호 개수입니다.
- `NEIGHBOR`: 직전 회차 번호의 이웃수 개수입니다.
- `PREV_`: 전전 회차의 지표입니다.

## 희귀 홀짝 전이

희귀 홀짝은 다음 패턴으로 정의합니다.

- `5:1`
- `1:5`
- `6:0`
- `0:6`

분석 리포트는 희귀 홀짝 직후 다음 회차 홀짝 분포를 별도로 출력합니다. 예측 점수에서는 직전 회차가 희귀 홀짝이고 후보 조합도 희귀 홀짝일 때 보수적인 감점을 적용합니다.

현재 데이터 기준으로 희귀 홀짝 직후 다음 회차도 희귀 홀짝인 사례가 존재하므로, 이 규칙은 하드 제외가 아니라 감점으로만 사용합니다.
