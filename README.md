# 로또 분석 PWA (인앱)

이 폴더에서 함께 분석한 결론을 쓰는 **라이트 데스크톱 대시보드형 PWA**입니다.
‘아파트 왓’ 실거래가 대시보드 컨셉을 로또 데이터에 맞춰 옮겼습니다.
당첨번호 예측은 불가능하다는 검증 결과를 정직하게 반영하고,
데이터가 지지하는 **유일한 전략 '1~12 회피'**(당첨 시 기대 수령액 ↑)를 중심으로 만들었습니다.

## 화면 (상단 내비 4개)

- **대시보드**: KPI 카드 5장 + 필터바(회차범위·1~12개수·홀짝·포함번호·비인기 토글) + 번호 출현 히트맵(클릭 시 필터) + 차트 3개(홀짝 막대 / 회차별 1등 당첨자 라인 / 번호대 G1~G5 도넛) + 정렬 가능한 당첨 내역 테이블. 모든 필터가 KPI·히트맵·차트·표에 동시 적용.
- **번호 추천**: 비인기(1~12 회피) 조합 생성. 고정/제외/연속회피 옵션.
- **백테스트**: 1~12 그룹별 실제 당첨자 + 시기별 신호 안정성 + 회차 조회.
- **인사이트**: 검증된 결론 카드.

## 구성

| 파일 | 역할 |
|---|---|
| `index.html` / `app.js` / `styles.css` | 앱 본체 (라이트 대시보드, SVG 차트 직접 렌더 — 외부 라이브러리 없음) |
| `data.js` | 빌드로 생성된 데이터(회차·통계·백테스트·회귀계수). **직접 수정 금지** |
| `manifest.webmanifest` / `sw.js` | PWA 설정·오프라인 캐시(네트워크 우선) |
| `icon-192.png` / `icon-512.png` | 앱 아이콘 |

상위 폴더의 빌드 스크립트:
- `build_app_data.py` — CSV → `app/data.js` 재생성 (회차 갱신 시 실행)
- `gen_icons.py` — 아이콘 PNG 재생성
- `serve.py` — 로컬/폰 접속용 서버

## 실행

```bash
# 1) 데이터 갱신(새 회차 추가 시)
python3 build_app_data.py

# 2) 서버 실행
python3 serve.py            # 기본 8000 포트
```

- PC 브라우저: `http://localhost:8000`
- 폰(같은 와이파이): 실행 시 출력되는 `http://<PC IP>:8000`

## 폰에서 앱처럼 쓰기 (홈 화면에 추가)

- **iPhone(Safari)**: 공유 → "홈 화면에 추가". 전체화면 앱처럼 실행됩니다.
- **Android(Chrome)**: 메뉴 → "앱 설치" / "홈 화면에 추가".

> 오프라인 캐시(서비스워커)는 보안 컨텍스트(localhost 또는 https)에서만 동작합니다.
> 폰에서 LAN IP(http)로 열면 화면은 잘 뜨지만 완전 오프라인 캐시는 제한될 수 있습니다.
> 항상 오프라인으로 쓰려면 https로 배포(예: GitHub Pages)하세요.

## 정직 고지

- 1등 확률은 어떤 조합이든 1/8,145,060으로 동일합니다.
- 이 앱은 적중률을 올리지 못합니다. '당첨됐을 때' 덜 나눠 갖는 기대값만 최적화합니다.
- 효과 크기는 작고(모델 R²≈2%) 최근 약해지는 추세입니다. 재미와 분석용으로 쓰세요.
