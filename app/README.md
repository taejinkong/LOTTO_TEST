# 로또 분석 PWA

동행복권 공식 당첨 데이터와 `integrated_rules_v2` 통합 점수 모델을 사용하는 정적 웹앱입니다.

## 화면

- **대시보드**: 회차·홀짝·포함 번호 필터, 번호 히트맵, 분포 차트, 당첨 내역
- **번호 생성**: 전체 규칙 통합 v2/페어+주기 v1 선택, 재현 시드, 조합 분산, 선택형 실험 필터
- **모델 검증**: 무작위 기대 기준을 포함한 30회 워크포워드 백테스트, 현재 점수와 규칙별 배제 위험
- **예측 원장**: 생성 결과를 조건·시드와 함께 SHA-256 영수증으로 잠금, 추첨 후 실제 1~5등 자동 판정, JSON 내보내기
- **인사이트**: 모델 해석과 한계

스마트폰에서는 핵심 지표가 2열로 재배치되고, 회차 목록은 카드형으로 전환됩니다.
주요 화면은 하단 고정 메뉴로 이동할 수 있으며 입력·버튼은 터치하기 쉬운 크기를 유지합니다.

웹 생성기는 `build_app_data.py`가 내보낸 기본 분포, 전이, 조건부 규칙, 개별 번호,
페어, 주기 점수와 희귀 홀짝 감점을 사용합니다.
CLI 생성기 `lotto_combo_generator.py`도 같은 `lotto_prediction.py` 규칙을 사용합니다.

합계·홀짝 극단·끝수 합계·3연속·번호대·직전 번호 제한은 기본 점수 모델에
강제로 포함하지 않습니다. 번호 생성 화면에서 직접 선택할 수 있는 실험 필터이며,
각 규칙이 과거 당첨조합과 무작위 후보를 얼마나 제외하는지 함께 표시합니다.

예측 원장은 서버가 아닌 현재 브라우저의 `localStorage`에 저장됩니다. 브라우저
데이터를 지우면 사라질 수 있으므로 중요한 기록은 원장 화면에서 JSON으로
내보내 보관해야 합니다.

## 데이터 갱신

```bash
python3 update_lotto_csv.py
```

새 공식 회차를 CSV에 추가한 뒤 `app/data.js`를 자동 재생성합니다. 앱만 다시 빌드하려면:

```bash
python3 build_app_data.py
```

로컬 확인은 저장소 루트에서 `python3 serve.py`를 실행한 뒤 `http://localhost:8000`을 엽니다.

모든 단일 조합의 1등 확률은 1/8,145,060으로 동일합니다. 점수는 후보 정렬 기준이며 당첨을 보장하지 않습니다.
