> 2026-07-23 업데이트: 앱 데이터는 606~1233회이며, 웹/CLI 기본 생성 모델은

> 결과는 초기 개발 당시 기록으로 보존합니다. 최신 30회 비교는

## 1. 목적

이 프로젝트는 대한민국 로또 6/45 1등 당첨번호 데이터를 바탕으로 과거 패턴을 분석하고, 후보 조합을 점수화하는 예측 보조 도구를 만드는 것을 목표로 한다.

단, 로또는 본질적으로 무작위 추첨이므로 이 도구는 정답을 보장하는 예측기가 아니라, 과거 통계와 조건부 패턴을 이용해 후보 조합을 해석하고 검증하는 분석 도구로 정의한다.

## 2. 데이터 기준

원본 데이터:

- `로또 회차별 당첨번호_20260620124844.xlsx`

분석용 CSV:

- `lotto_winners_2020_2026.csv`

현재 분석 범위:

- `606회~1228회`
- `2014-07-12~2026-06-13`
- 총 `623회`

엑셀에는 날짜 컬럼이 없으므로 날짜는 다음 공식으로 계산한다.

```text
1회 = 2002-12-07
회차 날짜 = 2002-12-07 + (회차 - 1) * 7일
```

검증된 주요 회차:

```text
606회  = 2014-07-12
1226회 = 2026-05-30
1227회 = 2026-06-06
1228회 = 2026-06-13
```

## 4. 예측 항목

현재 분석/예측에 반영된 주요 항목은 다음과 같다.

| 항목 | 설명 |
|---|---|
| 출현 횟수 | 번호별 전체 출현 빈도 |
| 최근 출현 횟수 | 최근 N회 기준 번호별 출현 빈도 |
| 출현주기 | 번호별 평균 출현 간격 대비 현재 미출현 정도 |
| 번호쌍 점수 | 후보 6개 안의 15개 번호쌍 과거/최근 출현 점수 |
| 끝수 통계 | 번호 마지막 자리 분포 |
| 공색/번호대 그룹 | `1~10`, `11~20`, `21~30`, `31~40`, `41~45` |
| 1~10 미출현 | 1~10 번호가 하나도 없는 패턴 |
| 홀짝 비율 | 홀수:짝수 비율 |
| 희귀 홀짝 전이 | `5:1`, `1:5`, `6:0`, `0:6` 이후 희귀 홀짝 재출현 감점 |
| 합계값 | 번호 6개의 총합 구간 |
| 첫 수 | 가장 작은 번호의 구간 |
| 고번호 개수 | `31~45` 번호 개수 |
| 간격수 | 정렬된 번호 사이의 평균 간격 |
| AC값 | 번호 간 차이의 다양성 |
| 연속번호 | 연속번호 쌍 개수 |
| 이월수 | 직전 회차와 같은 번호 개수 |
| 이웃수 | 직전 회차 번호의 ±1 번호 개수 |
| 조건부 연계 규칙 | `IF 이전 패턴 THEN 다음 패턴` 형태의 규칙 |

## 5. 공색/번호대 그룹 정의

공색 통계는 실제 공 색상이 아니라 번호 구간 그룹으로 해석한다.

| 그룹 | 범위 |
|---|---|
| G1 | 1~10 |
| G2 | 11~20 |
| G3 | 21~30 |
| G4 | 31~40 |
| G5 | 41~45 |

예:

```text
[24, 29, 30, 31, 35, 44]
COLOR = 0-0-3-2-1
```

## 6. 희귀 홀짝 전이 분석

희귀 홀짝:

```text
5:1
1:5
6:0
0:6
```

606회~1228회 기준:

```text
희귀 홀짝 직후 사례: 116회
다음 회차도 희귀 홀짝: 22회
비율: 19.0%
```

따라서 희귀 홀짝 이후 희귀 홀짝을 하드 제외하지 않고, 보수적 감점으로 반영한다.

## 7. 시나리오 구조

단일 점수 상위 조합만 뽑으면 특정 평균 패턴에 후보가 몰린다. 이를 보완하기 위해 시나리오별 후보군을 만든다.

| 시나리오 | 설명 |
|---|---|
| balanced | 합계 100~159, 연속번호 1쌍 이하 |
| high_sum | 합계 160 이상 |
| high_start | 첫 수 21 이상 |
| consecutive | 연속번호 2쌍 이상 |
| high_band | 31~45 번호 3개 이상 |

최종 Top 50 포트폴리오 쿼터:

```text
balanced    25개
high_sum     8개
high_band    7개
high_start   5개
consecutive  5개
```

## 8. 백테스트 기준

백테스트는 특정 회차를 예측 대상으로 두고, 그 이전 회차까지만 학습 데이터로 사용한다.

예:

```text
606회~1226회 데이터로 1227회 예측
```

평가 기준은 최종 Top 50 포트폴리오 안에서 다음 등수 조합이 몇 개 나오는지다.

```text
1등 = 당첨번호 6개 일치
2등 = 당첨번호 5개 + 보너스 번호 일치
3등 = 당첨번호 5개 일치
```

## 9. 1227회 백테스트 결과

조건:

```text
학습 데이터: 606회~1226회
예측 대상: 1227회
실제 번호: [1, 14, 16, 34, 41, 44]
보너스: 13
평가 조합 수: 8,145,060개
```

패턴 예측:

```text
AC: MATCH
연속번호: MATCH
첫 수: MATCH
1~10 개수: MATCH
간격: MISS
31~45 개수: MISS
홀짝: MISS
합계: MISS
```

패턴 적중:

```text
4/8
```

기존 단일 점수 방식:

```text
실제 조합 순위: 641,349 / 8,145,060
Top 50: NO
```

개선 포트폴리오 점수 방식:

```text
실제 조합 순위: 3,674,095 / 8,145,060
Top 50: NO
```

최종 Top 50 포트폴리오 등수 집계:

```text
1등 = 0개
2등 = 0개
3등 = 0개
```

결론:

```text
현재 구조는 1227회 기준으로 Top 50 안에 1~3등 조합을 만들지 못했다.
패턴 일부는 맞혔지만 실제 번호 조합 선택력은 부족하다.
번호쌍/출현주기 항목은 추가됐지만 현재 가중치로는 개선 효과가 확인되지 않았다.
```

## 10. 현재 한계

현재 도구의 주요 한계는 다음과 같다.

```text
패턴 예측과 실제 번호 선택 사이의 간극이 큼
번호쌍/출현주기 점수의 가중치가 검증되지 않음
단일 회차 백테스트만으로는 항목의 유효성을 판단하기 어려움
Top 50이라는 매우 좁은 후보군 안에 실제 1~3등을 넣는 것은 난도가 높음
```

## 11. 다음 개선 방향

우선순위가 높은 개선 방향:

```text
1. 다회차 백테스트
2. Top 50 내 1~3등 발생 횟수 누적 집계
3. 번호쌍/출현주기/패턴 점수 가중치 튜닝
4. 시나리오별 쿼터 자동 조정
5. 제외 규칙 백테스트
6. 보너스 번호 연계 추가
7. 전이 패턴 세분화
```

가장 중요한 다음 단계:

```text
예: 1200회~1228회 전체를 대상으로
각 회차마다 직전 회차까지만 학습
Top 50 포트폴리오 안의 1등/2등/3등 발생 개수 집계
```

이 결과가 있어야 어떤 예측 항목이 실제로 도움이 되는지 판단할 수 있다.

## 12. 실행 명령

엑셀을 CSV로 변환:

```bash

```

기본 분석:

```bash
python3 lotto_analyzer.py
```

```bash
python3 predict_lotto.py --per-scenario 5
```

1227회 백테스트:

```bash
python3 backtest_1228.py --target-round 1227 --mode full --top 50 --top-k 50 --per-scenario 50 --show 5
```

빠른 샘플 백테스트:

```bash
python3 backtest_1228.py --target-round 1227 --mode sample --sample-size 50000 --top 50 --top-k 50
```
